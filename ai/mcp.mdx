Umbra lets AI agents make **private** SPL payments without giving up custody. Two
pieces, both **non-custodial** (the agent holds its own key; signing and ZK
proving happen locally):

- **`@umbra-privacy/payments-mcp`** — an [MCP](https://modelcontextprotocol.io)
  server. Add one line to Claude Desktop / Claude Code and the agent gains
  `umbra_*` tools. This is the way to let an LLM pay conversationally.
- **`@umbra-privacy/agent-kit`** — the engine the MCP wraps. Import it to embed
  the same flows in your own backend or custom agent. See
  [Build it into your code](#build-it-into-your-code).

<Note>
The MCP is a thin adapter: it exposes the agent-kit's methods as tools and
reimplements no protocol logic. agent-kit is the brain; the MCP is the port that
plugs it into an AI agent. Pair either with the [Umbra skill](/ai/skill) so the
model knows the right flow.
</Note>

## Why non-custodial

Umbra's master seed — which derives every viewing key that can decrypt your
"private" balances — is itself derived from a wallet signature. Any party that
holds the key can see through the privacy, so a custodial hosted wallet is a
non-starter. Both paths keep the key and the ZK proof witness **on the agent's
host**: the agent signs locally, derives the seed locally, and proves locally
(~1–3s, Node-native). The relayer is only semi-trusted for burn submission. No
TEE, no custody server.

## Install

No install needed — run it with `npx`. Print the config snippet:

```bash
AGENT_KEYPAIR=/abs/path/agent.json UMBRA_NETWORK=mainnet npx -y @umbra-privacy/payments-mcp init
```

### Claude Desktop

Add the printed block to `claude_desktop_config.json` and restart:

```json
{
  "mcpServers": {
    "umbra-payments": {
      "command": "npx",
      "args": ["-y", "@umbra-privacy/payments-mcp"],
      "env": {
        "AGENT_KEYPAIR": "/abs/path/agent.json",
        "UMBRA_NETWORK": "mainnet",
        "RPC_URL": "https://mainnet.helius-rpc.com/?api-key=YOUR_KEY"
      }
    }
  }
}
```

### Claude Code

```bash
claude mcp add umbra-payments -s user \
  -e AGENT_KEYPAIR=/abs/path/agent.json \
  -e UMBRA_NETWORK=mainnet \
  -e RPC_URL="https://mainnet.helius-rpc.com/?api-key=YOUR_KEY" \
  -- npx -y @umbra-privacy/payments-mcp
```

<Tip>
Install the [Umbra skill](/ai/skill) alongside the MCP so the model follows the
right flow, the jobId/poll pattern, and handles the typed errors.
</Tip>

## Configuration

- `AGENT_KEYPAIR` (**required**) — absolute path to the agent's Solana keypair JSON.
- `UMBRA_NETWORK` — `mainnet` | `devnet` (default) | `localnet`.
- `RPC_URL` — Solana RPC. Use a paid endpoint (Helius/Triton/QuickNode) in
  production.
- `UMBRA_RPC_TRANSPORT` — `polling` (default) | `websocket`. Polling awaits
  confirmations + Arcium MPC callbacks over HTTP, so it works on **any** RPC —
  including the public mainnet endpoint whose WebSocket refuses subscriptions. Use
  `websocket` (lower latency) only with a WS-capable RPC, plus `RPC_WS`.
- `RPC_WS` — WebSocket RPC URL. Used only when `UMBRA_RPC_TRANSPORT=websocket`.
- `UMBRA_CAPS` — JSON spending caps, smallest units as strings, e.g.
  `{"perCall":{"<mint>":"10000000"},"perDay":{"<mint>":"50000000"}}`.
- `UMBRA_STATE` — local state file (idempotency + caps + claim watermark).
  Defaults to `~/.umbra/agent/<network>/<owner>/state.json` (`0600`, outside any
  project tree).
- `UMBRA_SCAN_STORE` — directory for the encrypted scan store (defaults next to
  `UMBRA_STATE`).
- `UMBRA_CLAIM_MODE` — `incremental` (default) | `from-genesis`.
- `UMBRA_OFFSETS` — JSON key offsets (set to the `newOffsets` from a key rotation).

<Warning>
The agent keypair is a hot wallet on the host. Keep its balance small and set
`UMBRA_CAPS` — the maximum loss from a compromised host is the funded balance,
bounded by the caps. Treat `UMBRA_STATE` like the keyfile; it is not encrypted at
rest (the only key would derive from the co-located keyfile).
</Warning>

## How an agent drives it

Amounts are **strings in smallest units** (USDC = 6 decimals → 5 USDC = `"5000000"`;
SOL/wSOL = 9).

<Steps>
  <Step title="Orient">
    `umbra_status` — registration, native SOL (`solLamports`), public + private
    balances, pending inbound notes. (Native SOL and wSOL are reported separately.)
  </Step>
  <Step title="Shield">
    `umbra_deposit` — public ATA → encrypted balance. Auto-registers first if needed.
  </Step>
  <Step title="Pay">
    `umbra_pay` with a stable `idempotencyKey` (e.g. an invoice id) — a retry with
    the same key returns `replayed:true` and never double-pays.
  </Step>
  <Step title="Receive / withdraw">
    Recipient: `umbra_receive`. Either side: `umbra_withdraw` to a public ATA.
  </Step>
</Steps>

Slow operations (MPC, ~30s–2min) return `{ jobId }` immediately — the agent polls
`umbra_job_status`, it does not blind-retry.

```text
umbra_status                              -> registered:false, public 100 USDC
umbra_deposit { mint, amount:"20000000" } -> { jobId } ; poll -> completed
umbra_pay { recipient, mint, amount:"5000000", idempotencyKey:"inv-42" }
                                          -> { jobId } ; poll -> { path:"registered-note" }
umbra_receive                             -> { jobId } ; poll -> { claimed:1 }   # recipient side
```

## Tools

Reads (fast): `umbra_status`, `umbra_supported_mints`, `umbra_job_status`.

Lifecycle: `umbra_register` (idempotent; money tools auto-register).

Money (slow → return a jobId):

- `umbra_deposit({ mint, amount })` — shield: public ATA → encrypted balance.
- `umbra_pay({ recipient, mint, amount, idempotencyKey?, publicFallback? })` —
  private transfer. Registered recipient → anonymous receiver-burnable note;
  unregistered → payout to their ATA (see below).
- `umbra_receive({ fromGenesis? })` — scan + claim inbound notes. Fast by default
  (persistent encrypted scan cursor + claim watermark); `fromGenesis:true`
  re-scans to recover any miss.
- `umbra_withdraw({ mint, amount })` — unshield: encrypted balance → public ATA.
- `umbra_convert({ mints })` — MXE-locked → Shared so a balance becomes spendable.

Recovery / compliance / rotation:

- `umbra_recover_staged({ mint, amount, destinationAta })` — reclaim funds staged
  by a dropped MPC callback.
- `umbra_export_viewing_key({ level, mint?, year?, month?, day? })` — scoped,
  read-only audit key for stealth-pool history. See [Compliance](/sdk/compliance-viewing-keys).
- `umbra_issue_grant({ grantee })` — authorize an auditor to re-encrypt
  encrypted-balance ciphertexts; returns a `nonce` (store it). `umbra_revoke_grant({ grantee, nonce })`.
- `umbra_rotate_keys({ keys, sweepMints? })` — rotate `userEncryption` / `mvk`.
  **Destructive**; sweep first; returns `newOffsets` to persist + restart with.

## Paying an unregistered recipient

A private transfer needs the recipient **registered** (so they can claim the
note). For an unregistered recipient:

- **Mainnet** — `umbra_pay` pays out to their public ATA via a self-burnable note.
- **Devnet / localnet** — that path is blocked by the network's MPC config, so
  `umbra_pay` throws `RECIPIENT_NOT_REGISTERED` and **moves no funds**. Either have
  them run `umbra_register` (then re-pay — stays private), or retry with
  `publicFallback:true` to deliver via a public SPL transfer (NOT private).

## Typed errors

Surface and act — don't loop: `INSUFFICIENT_SOL`, `INSUFFICIENT_FUNDS`,
`RECIPIENT_NOT_REGISTERED`, `MXE_LOCKED` (→ `umbra_convert`), `CAP_EXCEEDED`,
`BURN_FAILED`.

## Build it into your code

If you're writing code rather than driving an LLM, use the engine directly:

```bash
npm install @umbra-privacy/agent-kit @umbra-privacy/sdk@5.0.0-rc.6
```

```ts
import { createUmbra, loadSignerFromBytes } from "@umbra-privacy/agent-kit";

const umbra = await createUmbra({
  network: "mainnet",
  signer: await loadSignerFromBytes(secretKeyBytes), // local keypair — non-custodial
  rpcUrl: process.env.RPC_URL,
  caps: { perCall: { [USDC]: 10_000_000n }, perDay: { [USDC]: 100_000_000n } },
});

await umbra.status([USDC]);
await umbra.shield(USDC, 20_000_000n);
await umbra.transfer({ recipient, mint: USDC, amount: 5_000_000n, idempotencyKey: "inv-42" });
await umbra.receive();
await umbra.unshield(USDC, 5_000_000n);
```

`@umbra-privacy/sdk` is a **peer dependency** — install the exact version you pin
(the ZK assets are versioned in lockstep with the SDK). The engine is
network-agnostic; the same code runs on mainnet, devnet, and localnet. Catch
`AgentKitError` and branch on `.code` for the typed errors above.

## Networks

- **Supported mints** differ by network (real USDC/USDT/UMBRA/CASH on mainnet) —
  see [Supported Tokens](/supported-tokens).
- **Private pay to an unregistered recipient** is mainnet-only (see above).

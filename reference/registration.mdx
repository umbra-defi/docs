## getUserRegistrationFunction

**Import:** `@umbra-privacy/sdk/registration`

```typescript
function getUserRegistrationFunction(
  args: GetUserRegisrationFunctionArgs,
  deps?: GetUserRegistrationFunctionDeps,
): UserRegistrationFunction
```

Registers the caller on-chain. Registration is a multi-step idempotent flow that enables confidential token accounts (encrypted balances) and anonymous usage (mixer eligibility). Steps already completed in a previous session are skipped.

---

### GetUserRegisrationFunctionArgs

(name carries a known typo in the current RC — the second "t" is missing in the exported symbol)

- `client: IUmbraClient` - The configured client.

---

### GetUserRegistrationFunctionDeps

All fields are optional. Categories fall back to provider equivalents on `client`.

```typescript
interface GetUserRegistrationFunctionDeps {
  readonly rpc?: {
    readonly accountInfoProvider?: AccountInfoProviderFunction;
    readonly blockhashProvider?: GetLatestBlockhash;
    readonly transactionForwarder?: TransactionForwarder;
  };
  readonly keys?: {
    readonly tokenEncryptionX25519KeypairDeriver?: Curve25519KeypairGeneratorFunction;
    readonly masterViewingKeyEncryptingX25519KeypairDeriver?: Curve25519KeypairGeneratorFunction;
    readonly masterViewingKeyGenerator?: MasterViewingKeyDeriverFunction;
    readonly masterViewingKeyBlindingFactorGenerator?: MasterViewingKeyBlindingFactorDeriverFunction;
    readonly poseidonPrivateKeyGenerator?: PoseidonPrivateKeyDeriverFunction;
    readonly poseidonBlindingFactorGenerator?: PoseidonBlindingFactorDeriverFunction;
  };
  readonly crypto?: {
    readonly userCommitmentGenerator?: UserCommitmentGeneratorFunction;
  };
  /** Required when `anonymous: true`. Use `getUserRegistrationProver()` from `@umbra-privacy/sdk/zk-prover`. */
  readonly zkProver?: IZkProverForUserRegistration;
  /** Deeper overrides forwarded to the confidential-usage step executor. */
  readonly confidentialExecutorDeps?: GetRegisterUserForConfidentialUsageExecutorDeps;
  /** Deeper overrides forwarded to the anonymous-usage step executor (Rescue, Fiat-Shamir, polynomial commitments, etc.). */
  readonly anonymousExecutorDeps?: GetRegisterUserForAnonymousUsageV17ExecutorDeps;
}
```

The Rescue cipher / Fiat-Shamir / polynomial-commitment overrides live inside `anonymousExecutorDeps` — they are not top-level on `GetUserRegistrationFunctionDeps`. See `@umbra-privacy/sdk/registration` source for the exact `*ExecutorDeps` shapes.

---

### Returns

`UserRegistrationFunction`

```typescript
type UserRegistrationFunction = (
  options?: UserRegistrationOptions,
) => Promise<TransactionSignature[]>
```

Returns the signatures of all transactions submitted during the registration flow. Already-completed steps produce no transactions.

### UserRegistrationOptions

- `confidential?: boolean` - Register for confidential (encrypted) token account usage. Default: `true`.
- `anonymous?: boolean` - Register for anonymous (mixer) usage. Requires a ZK proof. Default: `true`.
- `hooks?: RegistrationHooks` - Per-step lifecycle hooks. Slots: `initUserAccount`, `registerX25519PublicKey`, `registerAnonymousUsage` (each `SkippableDirectTransactionStepHooks` or `SkippableMpcTransactionStepHooks`), plus top-level `onValidationStart/Complete`, `onAccountFetchStart/Complete`, `onKeyDerivationStart/Complete`, `onComplete`, `onError`.
- `optionalData?: UserRegistrationOptionalData` - Per-step 32-byte caller metadata. An **object** (not a single 32-byte value) with optional `OptionalData32` sub-fields, one per registration instruction: `accountInitialisation`, `registerX25519PublicKey`, `registerUserForAnonymousUsage`. Any omitted field defaults to 32 zero bytes.

---

### Errors

Throws `RegistrationError`. See [Errors](./errors#registrationerror) for all stages.

### Example

```typescript
import { getUserRegistrationFunction } from "@umbra-privacy/sdk/registration";
import { getUserRegistrationProver } from "@umbra-privacy/sdk/zk-prover";

const zkProver = getUserRegistrationProver();

const register = getUserRegistrationFunction({ client }, { zkProver });
const signatures = await register({ confidential: true, anonymous: true });
```

## getUserEncryptionKeyRotatorFunction

**Import:** `@umbra-privacy/sdk/account`

```typescript
function getUserEncryptionKeyRotatorFunction(
  args: { client: IUmbraClient },
  deps?: GetUserEncryptionKeyRotatorFunctionDeps,
): RotateUserEncryptionKeyFunction;
```

Rotates the X25519 public key stored in the on-chain user account. Synchronous, no MPC. Use this when the existing key may have been compromised.

### Returns

```typescript
type RotateUserEncryptionKeyFunction = (
  options?: RotateUserEncryptionKeyOptions,
) => Promise<TransactionSignature>;
```

Returns a single `TransactionSignature`. The returned function takes an optional `RotateUserEncryptionKeyOptions` (`optionalData?: OptionalData32`, `accountInfoCommitment?: "processed" | "confirmed" | "finalized"`, `hooks?: SimpleOperationHooks`).

## getMasterViewingKeyRotatorFunction

**Import:** `@umbra-privacy/sdk/account`

```typescript
function getMasterViewingKeyRotatorFunction(
  args: { client: IUmbraClient },
  deps: GetMasterViewingKeyRotatorFunctionDeps,
): RotateMasterViewingKeyFunction;
```

Rotates the Master Viewing Key (MVK) X25519 encryption key. **MPC + ZK** — also rotates the on-chain user commitment. `deps` is **required**: `GetMasterViewingKeyRotatorFunctionDeps` carries a required `zkProver: IZkProverForUserRegistration` (rotation cannot run without it), plus optional `rpc` overrides.

### Returns

```typescript
type RotateMasterViewingKeyFunction = (
  options?: RotateMasterViewingKeyOptions,
) => Promise<TransactionSignature>;
```

Returns a single `TransactionSignature`. The returned function takes an optional `RotateMasterViewingKeyOptions` (`optionalData?: OptionalData32`, `accountInfoCommitment?: "processed" | "confirmed" | "finalized"`, `microLamportsPerAcu?: MicroLamportsPerAcu`, `hooks?: SimpleOperationHooks`).

## getUserEntropySeedRotatorFunction

**Import:** `@umbra-privacy/sdk/account`

```typescript

  args: { client: IUmbraClient },
  deps?: GetUpdateRandomGenerationSeedFunctionDeps,

```

Rotates the random generation seed stored in the on-chain user account. This seed contributes to commitment randomness in Stealth Pool Note operations. Synchronous, no MPC.

### Deps

`GetUpdateRandomGenerationSeedFunctionDeps` — optional `rpc` overrides: `blockhashProvider?: GetLatestBlockhash`, `transactionForwarder?: TransactionForwarder`.

### Returns

```typescript
type UpdateRandomGenerationSeedFunction = (
  newSeed: Uint8Array,
  optionalData?: OptionalData32,
  hooks?: SimpleOperationHooks,
) => Promise<TransactionSignature>;
```

Positional arguments (not a single options object). Returns a single `TransactionSignature`.

- `newSeed: Uint8Array` — the 32-byte replacement seed.

## getStagedSolRecovererFunction

**Import:** `@umbra-privacy/sdk/account`

```typescript
function getStagedSolRecovererFunction(
  args: { client: IUmbraClient },
  deps?: GetClaimStagedSolFromPoolFunctionDeps,
): ClaimStagedSolFromPoolFunction;
```

Recovers wSOL that was staged in the protocol's pool custody during a failed MPC callback. Call this if a deposit / withdraw / convert callback never landed and you need to reclaim the staged SOL. Synchronous — no MPC, no ZK proof.

### Deps

`GetClaimStagedSolFromPoolFunctionDeps` — optional `rpc` overrides: `blockhashProvider?: GetLatestBlockhash`, `transactionForwarder?: TransactionForwarder`.

### Returns

```typescript
type ClaimStagedSolFromPoolFunction = (
  mint: Address,
  amount: U64,
  destination: Address,
  optionalData?: OptionalData32,
  hooks?: SimpleOperationHooks,
) => Promise<TransactionSignature>;
```

Positional arguments. Returns a single `TransactionSignature`.

- `mint: Address` — the mint of the failed operation (typically the wSOL mint).
- `amount: U64` — the staged amount to reclaim.
- `destination: Address` — the account to receive the recovered SOL.

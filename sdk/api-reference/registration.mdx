## `getUserRegistrationFunction`

```typescript
import { getUserRegistrationFunction } from "@umbra-privacy/sdk/registration";

function getUserRegistrationFunction(args: GetUserRegistrationFunctionArgs): UserRegistrationFunction
```

Factory function that returns a callable registration function.

### `GetUserRegistrationFunctionArgs`

<ParamField path="client" type="IUmbraClient" required>
  The Umbra client instance.
</ParamField>

### Returned Function

```typescript
async function register(options?: UserRegistrationOptions): Promise<TransactionSignature[]>
```

Performs up to two on-chain registration sub-steps (V5 collapsed account init into both sub-steps via `init_if_needed`). Checks current state before each and skips any already completed. Idempotent and safe to call on every app startup.

Returns an array of transaction signatures — one per sub-step that was executed. An empty array means everything was already in place.

### `UserRegistrationOptions`

<ParamField path="confidential" type="boolean" default="true">
  Register the X25519 token-encryption key for Shared-mode ETAs. Without this, deposits stay in MXE-only mode and balances cannot be queried locally.
</ParamField>

<ParamField path="anonymous" type="boolean" default="true">
  Register the Poseidon user commitment via MPC + ZK. Required to receive **receiver-burnable** Stealth Pool Notes. Can be used independently of `confidential`.
</ParamField>

<ParamField path="hooks.initUserAccount" type="SkippableDirectTransactionStepHooks">
  Lifecycle hooks for the synchronous account-init sub-step. Slots: `onPreSend`, `onPostSend`, `onSkipped` (fires when the account already exists).
</ParamField>

<ParamField path="hooks.registerX25519PublicKey" type="SkippableDirectTransactionStepHooks">
  Lifecycle hooks for the `register_user_for_confidential_usage` sub-step (X25519 key + account init combined via `init_if_needed`). Slots: `onPreSend`, `onPostSend`, `onSkipped`.
</ParamField>

<ParamField path="hooks.registerAnonymousUsage" type="SkippableMpcTransactionStepHooks">
  Lifecycle hooks for the MPC + ZK `register_user_for_anonymous_usage_v17` sub-step (user-commitment registration). Slots: `onPreSend`, `onPostSend`, `onSkipped`, plus MPC progress slots (`onMonitorStarted`, `onProgress`, `onFinalized`).
</ParamField>

## `getUserAccountQuerierFunction`

```typescript
import { getUserAccountQuerierFunction } from "@umbra-privacy/sdk/query";

function getUserAccountQuerierFunction(args: {
  client: IUmbraClient;
}): QueryUserAccountFunction
```

Factory function that returns a read-only account state query.

### Returned Function

```typescript
async function queryUserAccount(
  userAddress: Address,
): Promise<QueryUserAccountResult>
```

### `QueryUserAccountResult`

```typescript
type QueryUserAccountResult =
  | { state: "non_existent" }
  | { state: "exists"; data: EncryptedUserAccount };
```

**`EncryptedUserAccount` fields:**

<ResponseField name="isInitialised" type="boolean">
  Base account exists on-chain.
</ResponseField>

<ResponseField name="isUserAccountX25519KeyRegistered" type="boolean">
  X25519 key has been registered (Shared mode enabled).
</ResponseField>

<ResponseField name="isUserCommitmentRegistered" type="boolean">
  Poseidon commitment has been registered (anonymous transfers enabled).
</ResponseField>

<ResponseField name="isActiveForAnonymousUsage" type="boolean">
  Both X25519 and commitment are registered - account is fully ready.
</ResponseField>

<ResponseField name="x25519PublicKey" type="Uint8Array">
  The user's registered X25519 public key (32 bytes).
</ResponseField>

<ResponseField name="generationIndex" type="bigint">
  Monotonic counter used for nonce derivation. Incremented on each deposit/withdrawal.
</ResponseField>

## V5 registration flow

```
Sub-step A: register_user_for_confidential_usage   [if confidential: true]
  └─ init_if_needed: EncryptedUserAccount PDA
  └─ Derives X25519 public key from master seed
  └─ Stores on-chain in EncryptedUserAccount
  └─ Synchronous, no MPC

Sub-step B: register_user_for_anonymous_usage_v17  [if anonymous: true]
  └─ init_if_needed: EncryptedUserAccount PDA
  └─ Derives Poseidon commitment from master seed
  └─ Stages master viewing key material
  └─ Runs Fiat-Shamir + Groth16, queues Arcium MPC
  └─ MPC + ZK
```

Both sub-steps `init_if_needed` the user account, so either may run first. Both require the master seed; the wallet signing prompt fires the first time a sub-step needs it.

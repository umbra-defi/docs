## `getNetworkEncryptionToSharedEncryptionConverterFunction`

```typescript
import { getNetworkEncryptionToSharedEncryptionConverterFunction } from "@umbra-privacy/sdk/conversion";

function getNetworkEncryptionToSharedEncryptionConverterFunction(
  args: { client: IUmbraClient },
  deps?: GetConvertToSharedEncryptionFunctionDeps,
): ConvertToSharedEncryptionFunction;
```

Factory function that returns a callable conversion function.

### Args

<ParamField path="client" type="IUmbraClient" required>
  The Umbra client. Must have an X25519 key registered on-chain (via `register({ confidential: true })`).
</ParamField>

### Returned function

```typescript
async function convertToSharedEncryption(
  mints: readonly Address[],
  optionalData?: OptionalData32,
  hooks?: ConvertToSharedHooks,
  microLamportsPerAcu?: MicroLamportsPerAcu,
): Promise<ConvertToSharedEncryptionResult>;
```

Upgrades MXE-only ETAs to Shared mode. Processes each mint in the array, skipping any that are not eligible.

### Parameters

<ParamField path="mints" type="readonly Address[]" required>
  Mint addresses to convert. Each must have an initialised ETA in MXE-only mode.
</ParamField>

<ParamField path="optionalData" type="Uint8Array">
  32 bytes of optional metadata. **Must be pre-hashed / pre-encrypted.** Defaults to zero bytes.
</ParamField>

<ParamField path="hooks" type="ConvertToSharedHooks">
  Per-operation lifecycle hooks. Slots: `onValidationStart`, `onValidationComplete`, `onConversionStart` (per mint), `onConversionComplete` (per mint), `onComplete`, `onError`. Import type from `@umbra-privacy/sdk/conversion`.
</ParamField>

<ParamField path="microLamportsPerAcu" type="MicroLamportsPerAcu">
  Priority fee in micro-lamports per Arcium Compute Unit. Controls transaction speed under network load.
</ParamField>

### Return value

```typescript
type ConvertToSharedEncryptionResult = {
  converted: Map<Address, TransactionSignature>;
  skipped: Map<Address, ConvertToSharedEncryptionSkipReason>;
};

type ConvertToSharedEncryptionSkipReason =
  | "non_existent"             // No ETA exists for this mint.
  | "not_initialised"          // ETA exists but is not initialised.
  | "already_shared"           // ETA is already in Shared mode.
  | "balance_not_initialised"; // Arcium balance not yet initialised.
```

## `getMintEncryptionKeyRotatorFunction`

```typescript
import { getMintEncryptionKeyRotatorFunction } from "@umbra-privacy/sdk/account";

function getMintEncryptionKeyRotatorFunction(
  args: { client: IUmbraClient },
  deps?: GetRotateMintX25519EncryptionKeyFunctionDeps,
): RotateMintX25519EncryptionKeyFunction;
```

Rotates the X25519 encryption key for a specific mint's Shared-mode ETA. **MPC operation** — re-encrypts the on-chain balance under the new per-mint key. Use this when the per-mint key may have been compromised. The account must already be in Shared mode.

### Args

<ParamField path="client" type="IUmbraClient" required>
  The Umbra client.
</ParamField>

### Deps

- `accountInfoProvider?: AccountInfoProviderFunction`.
- `getLatestBlockhash?: GetLatestBlockhash`.
- `transactionForwarder?: TransactionForwarder`.
- `mintX25519KeypairGenerator?: MintX25519KeypairGeneratorFunction` — override the per-mint X25519 keypair derivation.

### Returned function

```typescript
async function rotateMintKey(args: {
  mint: Address;
  optionalData?: OptionalData32;
  hooks?: RotateMintKeyHooks;
}): Promise<RotateMintKeyResult>;
```

### Example

```typescript
import { getMintEncryptionKeyRotatorFunction } from "@umbra-privacy/sdk/account";

const rotate    = getMintEncryptionKeyRotatorFunction({ client });
const signature = await rotate({ mint: usdcMint });
```

## Eligibility check

For each mint in the input array, the SDK checks the token account's status bits:

- Account must be initialised (`IS_INITIALISED` bit set).
- Account must be in MXE mode (`IS_SHARED_MODE` bit unset).
- Arcium balance must be initialised (`IS_ARCIUM_BALANCE_INITIALISED` bit set).

Ineligible accounts are added to `result.skipped` with the appropriate reason. Eligible accounts are re-encrypted on Arcium MPC and updated to Shared mode.

## How conversion works

Each conversion follows the [dual-instruction pattern](/concepts/how-umbra-works#the-dual-instruction-pattern):

1. **Handler instruction** — queues an Arcium re-encryption computation.
2. **Arcium MPC** — decrypts the MXE-only balance and re-encrypts under both the MPC key and the user's X25519 key.
3. **Callback instruction** — sets the `IS_SHARED_MODE` bit on the ETA.

After conversion, the account is in Shared mode and future balance queries via `getEncryptedBalanceQuerierFunction` decrypt locally using the user's X25519 private key.

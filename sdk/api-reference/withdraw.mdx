## `getETAIntoATAWithdrawerFunction`

```typescript
import { getETAIntoATAWithdrawerFunction } from "@umbra-privacy/sdk/withdrawal";

function getETAIntoATAWithdrawerFunction(
  args: GetETAIntoATAWithdrawerFunctionArgs,
  deps?: GetETAIntoATAWithdrawerFunctionDeps,
): ETAIntoATAWithdrawerFunction
```

Factory function that returns a callable withdrawal function.

### `GetETAIntoATAWithdrawerFunctionArgs`

<ParamField path="client" type="IUmbraClient" required>
  The Umbra client instance.
</ParamField>

### Returned Function

```typescript
async function withdraw(
  destinationAddress: Address,
  mint: Address,
  amount: bigint,
  options?: DirectWithdrawOptions,
): Promise<WithdrawResult>
```

Moves tokens from the signer's ETA back to the destination's ATA.

Follows the [dual-instruction pattern](/concepts/how-umbra-works#the-dual-instruction-pattern) - waits for both the handler transaction and the Arcium MPC callback transaction to confirm before resolving. Total wall-clock time is typically a few seconds.

### Parameters

<ParamField path="destinationAddress" type="Address" required>
  The Solana address to receive the withdrawn tokens. This is typically `client.signer.address`.
</ParamField>

<ParamField path="mint" type="Address" required>
  SPL or Token-2022 mint address. Must match a mint for which the signer has an existing ETA.
</ParamField>

<ParamField path="amount" type="bigint" required>
  Amount to withdraw in native token units. Must not exceed the current ETA balance.
</ParamField>

<ParamField path="options.optionalData" type="OptionalData32">
  32 bytes of opaque caller metadata. **Pre-hashed only.** Default 32 zero bytes.
</ParamField>

<ParamField path="options.accountInfoCommitment" type="Commitment" default='"confirmed"'>
  Commitment level for RPC account reads.
</ParamField>

<ParamField path="options.computationOffset" type="U64">
  Optional Arcium computation-account PDA offset. Defaults to a fresh random `U64` per call.
</ParamField>

<ParamField path="options.mpcCallbackDataOffset" type="U128">
  Optional MpcCallbackData PDA offset. Defaults to a fresh random `U128` per call.
</ParamField>

<ParamField path="options.feeVaultOffset" type="U128" default="0n">
  Fee-vault PDA offset. The fee-vault path is currently unused for public-ATA withdrawals — leave at `0n` (sentinel: no fee-vault interaction).
</ParamField>

<ParamField path="options.destinationProgram" type="Address">
  Observer-CPI destination program. Default: System Program (sentinel: no observer CPI).
</ParamField>

<ParamField path="options.hooks" type="WithdrawalHooks">
  Per-phase lifecycle hooks. See [Transaction Callbacks](/sdk/understanding-the-sdk/callbacks).
</ParamField>

<Note>
V4 fields `priorityFees`, `awaitCallback`, `skipPreflight`, `maxRetries`, `epochInfoCommitment` do not exist on the V5 withdrawer.
</Note>

### Return Value

Returns `Promise<WithdrawResult>`:

```typescript
interface WithdrawResult {
  readonly signatures: readonly TransactionSignature[];
  readonly queueSignature: TransactionSignature;
  readonly callback?:
    | { readonly status: "finalized"; readonly signature?: TransactionSignature; readonly elapsedMs: number }
    | { readonly status: "pruned";    readonly elapsedMs: number }
    | { readonly status: "timed-out"; readonly elapsedMs: number };
  readonly rentClaim?:
    | { readonly claimed: true;  readonly signature: TransactionSignature }
    | { readonly claimed: false; readonly reason: string };
}
```

---

## Withdrawal Destination

Tokens are sent to the ATA associated with `destinationAddress` for the given mint. If that ATA does not exist, the transaction will fail. To send to a different address anonymously, use the [Stealth Pool flow](/sdk/mixer/overview).

---

## MPC Timing

The withdrawal uses the dual-instruction pattern:

1. Handler instruction - sent and confirmed immediately
2. Arcium MPC computation - runs off-chain (typically 1-5 seconds)
3. Callback instruction - sent and confirmed by the SDK after MPC completes

The SDK's `await withdraw(...)` call spans all three steps.

## Error Class

Throws `EncryptedWithdrawalError` on failure.

```typescript
import { isEncryptedWithdrawalError } from "@umbra-privacy/sdk/errors";

try {
  await withdraw(client.signer.address, mint, amount);
} catch (err) {
  if (isEncryptedWithdrawalError(err)) {
    console.error("Stage:", err.stage, "Message:", err.message);
  }
}
```

See [Error Handling](/reference/errors) for the full list of stages.
